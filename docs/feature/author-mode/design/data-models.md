**Impact**: EF Core migration required to update the `Status` column. Currently stored as string in PostgreSQL â€” `"Archived"` will be the new valid value.

| Field | Type | Nullable | Description |
|---|---|---|---|
| `PreviousStatus` | `PostStatus` | Yes | Stored when transitioning to `Archived`. Cleared on `Restore`. Required to implement US-12 (restore to prior state). |

The `BlogPost` aggregate already has:

No other domain fields are added. The domain boundary does not expand beyond post lifecycle management.

### 1.3 New Domain Methods on `BlogPost`

| Method | Precondition | State Transition |
|---|---|---|
| `Archive(DateTime now)` | Status != Archived | Status â†’ Archived, PreviousStatus = old Status, UpdatedAt = now |
| `Restore(DateTime now)` | Status == Archived | Status = PreviousStatus, PreviousStatus = null, UpdatedAt = now |

Slug immutability enforcement point: `BlogPost.Publish()` should lock the slug. If this is not yet enforced at the domain level, it must be added here. The constraint is: once `PublishedAt` is set, `UpdateTitle` must not regenerate the slug.

## 3. Astro Session Schema

The Astro session is stored as a JSON object in Upstash Redis. Key: generated by the Astro Sessions driver. TTL: 24 hours (configurable via Astro session config).

```json
{
  "isAdmin": true,
  "email": "christian.borrello@gmail.com",
  "name": "Christian Borrello",
  "avatarUrl": "https://...",
  "jwtToken": "<signed-JWT-from-backend>",
  "jwtExpiresAt": "2026-03-14T15:00:00Z"
}
```

The `jwtToken` is a standard JWT issued by the .NET backend (using the existing `JwtTokenGenerator` infrastructure). Its TTL is 60 minutes by default (configurable via `Jwt:ExpiryInMinutes`). When the JWT expires, the next Action call returns 401, which the middleware interprets as a session renewal requirement â†’ redirect to `/admin/login`.

## 4. Admin Token Schema (Redis â€” Short-Lived)

Used only during the OAuth callback handshake. Keys are prefixed `admin_token:` and have a 5-minute TTL.

```
Key:   admin_token:<nonce_guid>
Value: { "email": "...", "name": "...", "avatarUrl": "..." }
TTL:   300 seconds (5 minutes)
```

The JWT itself (signed by `OAUTH_ADMIN_JWT_SECRET`) contains the `nonce_guid` as a claim. On `POST /api/auth/admin/verify-token`:
1. JWT signature and TTL are validated.
2. `nonce_guid` is extracted from JWT claims.
3. Redis key `admin_token:<nonce_guid>` is looked up.
4. If found: data returned, key deleted (single-use).
5. If not found: 401 (already used or expired).

## 5. API Contract Changes

### 5.1 `PostResponse` â€” Extended

The existing `PostResponse` record already includes `Status`. No changes needed â€” `"Archived"` will appear as the string value when a post is archived.

For admin list (`GET /api/admin/posts`): returns all statuses including `Archived`. Already scoped under `.RequireAuthorization()`.

For public browse (`GET /api/posts`): `BrowsePublishedPosts` filters to `Published` only. Archived posts are invisible to readers. No change needed here â€” the use case already filters by status.

### 5.2 New Request Bodies

**`PATCH /api/posts/{id}/archive`**: No request body. The operation is unambiguous.

**`PATCH /api/posts/{id}/restore`**: No request body. The operation reads `PreviousStatus` from the domain.

**`POST /api/auth/admin/verify-token`**:
```json
{ "token": "<short-lived-JWT>" }
```

Response:
```json
{
  "email": "christian.borrello@gmail.com",
  "name": "Christian Borrello",
  "avatarUrl": "https://...",
  "jwtToken": "<long-lived-JWT>",
  "jwtExpiresAt": "2026-03-14T15:00:00Z"
}
```

### 5.3 Cover Image in Post Create/Edit

`CreatePostRequest` and `EditPostRequest` already have a `FeaturedImageUrl` field (nullable string) in `PostResponse`. The API contracts for create and edit need to accept `coverImageUrl` (nullable) in the request body, so the frontend can pass the ImageKit URL after upload.

Current `CreatePostRequest` does not include `FeaturedImageUrl`. This must be added.

Updated request contracts:
- `CreatePostRequest`: add `string? CoverImageUrl = null`
- `EditPostRequest`: add `string? CoverImageUrl = null`
- `CreatePost` use case: accept and set `FeaturedImageUrl` on the `BlogPost`
- `EditPost` use case: accept and update `FeaturedImageUrl`
